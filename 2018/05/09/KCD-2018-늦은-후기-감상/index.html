KCD 2018 늦은 후기 / 감상

Posted on 2018-05-09 | Edited on 2018-05-10

KCD 2018 (Korea Community Day 2018)

지난 2월 24일 토요일에 KCD 2018에 다녀왔다. 갔다 온 뒤로는 시기가 시기였던지라 입학식이며 뭐며 이것저것 바빠서 행사에 대해서는 완전 잊고 있었는데, 메모를 정리하던 중 행사장에서 강연 중에 메모했던 것들이 보였다. 그대로 썩히기는 아깝고, 블로그 글도 써볼 겸 해서 그 때 적어두었던 메모를 조금 가공하여 적어볼까 한다.

개인적으로 그 때 어떤 생각을 하면서 강연을 들었고, 거진 2개월 후인 지금 다시 봤을 때의 감상을 정리한 것이다. 단지 개인적인 감상일 뿐이다.

(지금 다시 행사 사이트에 들어가 보니, 발표자료들이 대부분 업로드되어 있었다. 참고하면 좋을 듯하다. http://kcd2018.festa.io/)

차곡차곡 쉽게 알아가는 Elasticsearch와 Node.js - 황희정(자바카페)

DB에 단순히 쿼리를 날리는 기존의 검색방식의 한계를 설명하면서, Elasticsearch에 대해서 설명해주셨다. Elasticsearch의 특징, 데이터를 색인하는 과정, 유저 질의를 처리하는 과정 등을 간단하게나마 알 수 있었다. 그리고 “한글로 된” 영화 제목을 검색하는 간단한 프로젝트를 보여주고, Nodejs에 Integrate할 때 주의할 점 등으로 강연이 마무리되었던 것으로 기억한다.

좋았던 것은, 강의 전반에 걸쳐 한글 질의를 분석, 처리하는 과정을 중심으로 설명해주셨던 점이다. 한글 질의를 분석하는 방법은 영어로 된 Documentation에서는 절대로 알 수 없고, 국문으로 된 강좌나 책을 봐야만 알 수 있었을 것이다. 또한, 한글 분석기 이야기가 잠깐 나왔는데, 커뮤니티에서 만든 한글 분석기가 제법 여러 개 있다는 것과, 생각보다 본격적이라는 점이 재미있었다. “아버지가 방에 들어가신다” 문장을 예시로 들면서 각 한글 분석기가 어떤 식으로 문장을 쪼개는지 잠시 보여주셨는데, 명사, 조사, 동사 단위로 쪼개서 보여줄 수 있다는 것이 기억에 남는다.

마지막으로, elastic사에 대해 제대로 알게 된 계기가 이 강연인데, Elastic Stack이라는 기술 스택이 따로 존재한다는 것도 알게 되었다. Elasticsearch만 알고 있었지만, Kibana, Logstash부터 시작하여 elastic사의 제품이 생각 외로 많고, Documentation도 생각보다 잘 되어 있다는 것을 알게 되었다. 앞으로 데이터 분석, 딥러닝 쪽을 공부해보는 것이 목표인데, 나중에 elastic사의 제품을 만나게 된다면 어느 정도 익숙한 느낌이 들지 않을까 생각해본다. 뭐, 미래 이야기가 아니더라도, 당장 스타트업에서 하고 있는 프로젝트에 검색엔진을 도입할 필요가 생겼는데, 이번 기회에 이걸 한번 써 볼까 생각 중이다. 얻어갈 것이 많았던 강연이었다.

IntelliJ IDEA만으로 개발하기 - 이동욱 (JetBrains Korea User Group)

백엔드용으로 Eclipse를 쓰지만, 프론트엔드 개발용으로는 적합하지 않기 때문에 보조로 서브라임을 쓰고, DB관리용 툴 하나, Git 관리용으로 Sourcetree 등등, 배울 툴들이 너무 많다.

…이런 이야기로 시작하면서 IDEA에서 모든 것을 끝내는 모습을 보여주셨다.

여러 멋진 기능들을 보여주셨지만, 특히 Tasks 기능이 정말 마음에 들었다. IDEA 내에서 JIRA, Redmine, Trello같은 이슈트래커 서비스에 연동 설정해 두면, 그 이후로는 이슈트래커 사이트를 들어갈 필요 없이, 브랜치도 티켓 이름으로 다 만들어주고, 태스크 끝나면 티켓 없애고 지워준다. 이런 것이야말로 개발 효율을 최대한으로 끌어올릴 수 있는 방법이 아닐까 생각이 들었다.

또한 커밋 메시지 자동설정, 커밋할때 SonarLint 자동실행 등 많은 기능들이 기본으로 들어있었고, 서드파티 플러그인의 풀도 넓어서 GitFlow 플러그인 등은 손쉽게 찾을 수 있었다.

JetBrains사의 다른 제품인 Upsource도 잠깐 보여주셨는데, 코드리뷰 협업 툴인 듯했다. IDEA에서 코딩 중에 Upsource에서 변경된 내용이 오른쪽 밑에 실시간으로 푸시 알림이 뜨는 기능이 기억에 남는다.

IDE 하나로 모든 것을 해결 가능하다는 것은, 앱을 여러 개 설치하고, 창을 여러 개 넘나드는 수고를 줄일 수 있을 뿐만 아니라, 새 툴마다 단축키를 외울 필요가 없어진다는 것이기도 하다. 거의 모든 JetBrains 제품들이 단축키를 공유하고 있으므로, 하나만 제대로 배워두면 다른 언어를 개발할 때도 반은 먹고 들어가는 셈이다.

마지막 부분에서는 직접 만드신 IDEA 플러그인 ‘Translator’를 잠깐 소개해 주셨는데, 변수명이나 함수명 등을 지을 때 한글로 단어를 쓰고 키 조합을 누르면 번역하여 변수명이나 함수명을 만들어 주는 플러그인이었다. (기억으로는 camelCase까지 적용되는 등 굉장히 본격적이였던 것 같다. 단순히 한글 단어를 번역해서 영어로 바꿔 주는 플러그인이 아니였던 것이 기억에 남는다).

강연만 놓고 본다면 프로그래밍 언어나 알고리즘 같이 ‘프로그래머’스러운 내용을 다루는 강연은 아니었지만, 개발의 효율성이라는 것에 대해 다시 생각해보게 했던 강연이었다. 그리고 무엇보다도 나를 JetBrains빠로 만들어 준 강연이기도 하다. 몇 달 전까지만 해도 개발 환경이 제대로 정립되지 않았었지만, 지금은 JetBrains 제품들 몇 개와 VSCode만으로 어찌어찌 개발해나가고 있다.

C++ 프로젝트의 자동 빌드화 - 박동하(C++ Korea)

CI에 대해 잠깐 소개하고, Travis CI를 이용해 C++ 프로젝트를 자동으로 빌드, 배포하는 과정을 설명해주셨다.

강연 제목이 “C++ 프로젝트의 자동 빌드화”이긴 했지만, 특별히 C++ 언어에 특화된 강연은 아니였고, CI를 주제로 하는 내용으로 구성되어 있었다.

특히 초반에 CI에 대해 소개할 때 CI의 중요성에 대해 말씀하셨던 것이 기억에 남는데, 프로젝트 페이지에 build passing 떠 있으면 프로젝트에 확신을 줄 수 있다는 말이 어느 정도 공감이 되었고, 앞으로 만드는 오픈소스 프로젝트에 CI를 적극적으로 도입해봐야겠다는 생각이 들었다.

그 이후로는 Github 레포에 Travis CI 연동시키는 법, .travis.yml 작성법, 마지막으로 배포하는 법까지, 프로젝트 하나에 바로 써먹을 수 있을 정도로 자세하게 설명해주셨다.

(작성중!)
